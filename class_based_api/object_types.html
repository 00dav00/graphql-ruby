Object Types

Object types describe structures that have their own pieces of data called fields. Fields are strongly typed and can point to scalars, enums, other objects, interfaces, or unions. Hereâ€™s an example of a basic User type in GraphQL-Ruby:

class UserType < GraphQL::Schema::Object
  field :handle, String, null: false
  field :email, String, null: true
  field :is_admin, Boolean, null: false
end

You may also see objects expressed in the GraphQL Schema Definition Language (SDL):

type User {
  handle: String!
  email: String
  isAdmin: String!
}

User has three fields: handle, email and isAdmin:

  • handle returns a string and will never return null
  • email also returns a string, but may return null
  • isAdmin returns true or false, but never null

Notice that field :is_admin in Ruby became isAdmin in GraphQL.
