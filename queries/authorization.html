Queries — Authorization

GraphQL offers a few ways to ensure that clients access data according to their permissions.

  • Query analyzers can assert that the query is valid before running it.
  • Resolve wrappers can assert that returned objects are permitted to a given user.

Query Analyzers

A query analyzer visits each field in the query before the query is executed. It can accumulate data during the visits, then return a value. If the returned value is a GraphQL::AnalysisError (or an array of those errors), the query won’t be executed and the error will be returned to the user. You can use this feature to assert that queries are permitted before running them!

Query analyzers reuse concepts from Array#reduce, so let’s briefly revisit how that method works:

1items = [1, 2, 3, 4, 5]
2initial_value = 0
3reduce_result = items.reduce(initial_value) { |memo, item| memo + item }
4final_value = "Sum: #{reduce_result}"
5puts final_value
6# Sum: 15
  • reduce accepts an initial value and a callback (as a block)
  • The callback receives the reduce state (memo) and each item of the array (item)
  • For each call to the callback, the return value is the new state and it will be provided to the next call to the callback
  • When each item has been visited, the last value of the callback state (the last memo value) is returned
  • Then, you can use the reduced value in your application

A query analyzer has the same basic parts. Here’s the scaffold for an analyzer:

 1class MyQueryAnalyzer
 2  # Called before the visit.
 3  # Returns the initial value for `memo`
 4  def initial_value(query)
 5  end
 6
 7  # This is like the `reduce` callback.
 8  # The return value is passed to the next call as `memo`
 9  def call(memo, visit_type, irep_node)
10  end
11
12  # Called when we're done the whole visit.
13  # The return value may be a GraphQL::AnalysisError (or an array of them).
14  # Or, you can use this hook to write to a log, etc
15  def final_value(memo)
16  end
17end
  • #initial_value is a chance to initialize the state for your analysis. For example, you can return a hash with keys for the query, schema, and any other values you want to store.
  • #call is called for each node in the query. memo is the analyzer state. visit_type is either :enter or :leave. irep_node is the GraphQL::InternalRepresentation::Node for the current field in the query. (It is like item in the Array#reduce callback.)
  • #final_value is called after the visit. It provides a chance to write to your log or return a GraphQL::AnalysisError to halt query execution.

Query analyzers are added to the schema with query_analyzer, for example:

1MySchema = GraphQL::Schema.define do
2  query_analyzer MyQueryAnalyzer.new
3end

GraphQL’s max_depth and max_complexity are implemented with query analyzers, you can see those for reference:

  • GraphQL::Analysis::QueryDepth
  • GraphQL::Analysis::QueryComplexity

Resolve Wrapper

Sometimes, you can only check permissions when you have the actual object. Let’s say you’re exposing documents in your API:

1field :documents, types[DocumentType] do
2  resolve ->(obj, args, ctx) {
3    documents = obj.documents
4    # sort, filter, etc
5    # return the documents:
6    documents
7  }
8end

You can “wrap” this resolve function to assert that the documents are ok for the current user:

 1# Take a resolve function and call it.
 2# Then, check that the result includes permitted records _only_.
 3# @return [Proc] a new resolve function that checks the return values
 4def assert_allowed_documents(resolve_func)
 5  ->(obj, args, ctx) {
 6    documents = resolve_func.call(obj, args, ctx)
 7    current_user = ctx[:current_user]
 8
 9    if documents.all? { |d| current_user.can_view?(d) }
10      documents
11    else
12      nil
13    end
14  }
15end
16
17# ...
18
19field :documents, types[DocumentType] do
20  # wrap the resolve function with your assertion
21  resolve assert_allowed_documents(->(obj, args, ctx) {
22    # ...
23  })
24end

This way, you can “catch” the returned value before giving it to a client.

This approach can be further parameterized by implementing it as a class, for example:

 1# Assert that the current user has `permission` on the return value of `block`
 2class PermissionAssertion
 3  # Get a permission level and the "inner" resolve function
 4  def initialize(permission, resolve_func)
 5    @permission = permission
 6    @resolve_func = resolve_func
 7  end
 8
 9  # GraphQL will call this, so delegate to the "inner" resolve function
10  # and check the return value
11  def call(obj, args, ctx)
12    value = @resolve_func.call(obj, args, ctx)
13    current_user = ctx[:current_user]
14    if current_user.can?(@permission, value)
15      value
16    else
17      nil
18    end
19  end
20end
21
22# ...
23
24# Apply this class to the resolve function:
25field :documents, types[DocumentType] do
26  resolve PermissionAssertion.new(:view, ->(obj, args, ctx) {
27    # ...
28  })
29end
