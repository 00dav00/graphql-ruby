Schema — Lazy Execution

With lazy execution, you can optimize access to external services (such as databases) by making batched calls. Building a lazy loader has three steps:

  • Define a lazy-loading class with one method for loading & returning a value
  • Connect it to your schema with GraphQL::Schema#lazy_resolve
  • In resolve functions, return instances of the lazy-loading class

graphql-batch provides a powerful, flexible toolkit for lazy resolution with GraphQL.

Lazy resolution can be instrumented.

Example: Batched Find

Here’s a way to find many objects by ID using one database call, preventing N+1 queries.

  1. Lazy-loading class which finds models by ID.
 1class LazyFindPerson
 2  def initialize(query_ctx, person_id)
 3    @person_id = person_id
 4    # Initialize the loading state for this query,
 5    # or get the previously-initiated state
 6    @lazy_state = query_ctx[:lazy_find_person] ||= {
 7      pending_ids: [],
 8      loaded_ids: {},
 9    }
10    # Register this ID to be loaded later:
11    @lazy_state[:pending_ids] << person_id
12  end
13
14  # Return the loaded record, hitting the database if needed
15  def person
16    # Check if the record was already loaded:
17    loaded_record = @lazy_state[:loaded_ids][@person_id]
18    if loaded_record
19      # The pending IDs were already loaded,
20      # so return the result of that previous load
21      loaded_record
22    else
23      # The record hasn't been loaded yet, so
24      # hit the database with all pending IDs
25      pending_ids = @lazy_state[:pending_ids]
26      people = Person.where(id: pending_ids)
27      people.each { |person| @lazy_state[:loaded_ids][person.id] = person }
28      # Now, get the matching person from the loaded result:
29      @lazy_state[:loaded_ids][@person_id]
30    end
31  end
  1. Connect the lazy resolve method
1MySchema = GraphQL::Schema.define do
2  # ...
3  lazy_resolve(LazyFindPerson, :person)
4end
  1. Return lazy objects from resolve
1field :author, PersonType do
2  resolve ->(obj, args, ctx) {
3    LazyFindPerson(ctx, obj.author_id)
4  }
5end

Now, calls to author will use batched database access. For example, this query:

1{
2  p1: post(id: 1) { author { name } }
3  p2: post(id: 2) { author { name } }
4  p3: post(id: 3) { author { name } }
5}

Will only make one query to load the author values.

The example above is simple and has some shortcomings. Consider the graphql-batch gem for a robust solution to batched resolution.
