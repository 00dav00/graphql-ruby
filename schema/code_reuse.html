Schema — Code Reuse

Here are a few techniques for code reuse with graphql-ruby:

  • Resolver classes
  • Dynamically defining types
  • Functional composition and resolve

Besides reducing duplicate code, these approaches also allow you to test parts of your schema in isolation.

Resolver classes

(Originally described by AlphaSights at https://m.alphasights.com/graphql-ruby-clean-up-your-query-type-d7ab05a47084.)

Many examples use a Proc literal for a field’s resolve, for example:

1field :name, types.String do
2  # Resolve taking a Proc literal:
3  resolve ->(obj, args, ctx) { obj.name }
4end

However, you can pass any object to resolve(...), so long as it responds to .call(obj, args, ctx).

This means you can pass in a module:

 1module NameResolver
 2  def self.call(obj, args, ctx)
 3    obj.name
 4  end
 5end
 6
 7# use `NameResolver` instead of a proc
 8field :name, types.String do
 9  resolve(NameResolver)
10end
11# `resolve: resolver` is equivalent to `do resolve(resolver) end`
12# so this is the same:
13field :name, types.String, resolve: NameResolver

Or, you can pass an instance of a class:

 1class MethodCallResolver
 2  def initialize(method_name)
 3    @method_name = method_name
 4  end
 5
 6  def call(obj, args, ctx)
 7    obj.public_send(@method_name)
 8  end
 9end
10
11# ...
12field :name, types.String do
13  resolve(MethodCallResolver.new(:name))
14end

Or, you can generate the resolve Proc dynamically:

 1# @return [Proc] A resolve proc which calls `method_name` on `obj`
 2def resolve_with_method(method_name)
 3  ->(obj, args, ctx) { obj.public_send(method_name) }
 4end
 5
 6# ...
 7
 8field :name, types.String do
 9  resolve(resolve_with_method(:name))
10end

Dynamically defining types

Many examples show how to use .define and store the result in a Ruby constant:

1PostType = GraphQL::ObjectType.define do ... end

However, you can call .define anytime and store the result anywhere. For example, you can define a method which creates types:

 1# @return [GraphQL::ObjectType] a type derived from `model_class`
 2def create_type(model_class)
 3  GraphQL::ObjectType.define do
 4    name(model_class.name)
 5    description("Generated programmatically from model: #{model_class.name}")
 6    # Make a field for each column:
 7    model_class.columns.each do |column|
 8      field(column.name, convert_type(column.type))
 9    end
10  end
11end
12
13# @return [GraphQL::BaseType] a GraphQL type for `database_type`
14def convert_type(database_type)
15  # ...
16end

You can also define fields for associated objects. You’ll need a way to access them programmatically.

 1# Hash<Model => GraphQL::ObjectType>
 2MODEL_TO_TYPE = {}
 3
 4def create_type(model_class)
 5  # ...
 6  GraphQL::ObjectType.define do
 7    # ...
 8    # Make a field for associations
 9    model_class.associations.each do |association|
10      # The proc will be eval'd later - by that time, there will be a type in the lookup hash
11      field(association.name, -> { MODEL_TO_TYPE[association.associated_model] })
12    end
13  end
14end
15
16all_models_in_application.each { |model_class| MODEL_TO_TYPE[model_class] = create_type(model_class) }

There is one caveat to using .define. The block is called with instance_eval, so self is a definition proxy, not the outer self. For this reason, you may need to assign values to local variables, then use them in .define. (.define has access to the local scope, but not the outer self.)

 1class DynamicTypeDefinition
 2  attr_reader :model
 3  def initialize(model)
 4    @model = model
 5  end
 6
 7  def to_graphql_type
 8    # This doesn't work because `model` is actually `self.model`, which doesn't work inside `.define`
 9    # GraphQL::ObjectType.define do
10    #   name(model.name)
11    # end
12    #
13    # Instead, assign a local variable first:
14    model_name = model.name
15    GraphQL::ObjectType.define do
16      name(model_name)
17    end
18    # 👌
19  end
20end

Functional composition & resolve

You can modify procs by wrapping them in other procs. This is a simple way to combine elements for a resolve function.

For example, you can wrap a proc with authorization logic:

 1module Auth
 2  # Wrap resolve_proc in a check that `ctx[:current_user].can_read?(item_name)`
 3  #
 4  # @yield [obj, args, ctx] Field resolution parameters
 5  # @yieldreturn [Object] The return value for this field
 6  # @return [Proc] the passed-in block, modified to check for `can_read?(item_name)`
 7  def self.can_read(item_name, &block)
 8    ->(obj, args, ctx) do
 9      if ctx[:current_user].can_read?(item_name)
10        # continue to the next call:
11        block.call(obj, args, ctx)
12      else
13        nil
14      end
15    end
16  end
17end
18
19# ...
20
21QueryType = GraphQL::ObjectType.define do
22  field :findPost, PostType do
23    argument :id, !types.Int
24    resolve(Auth.can_read(:post) do |obj, args, ctx|
25      Post.find(args[:id])
26    end)
27  end
28end

Now, the inner proc will only be called if the outer proc calls it.
