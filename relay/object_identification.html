Relay — Object Identification

Relay uses global object identification support some of its features:

  • Caching: Unique IDs are used as primary keys in Relay’s client-side cache.
  • Refetching: Relay uses unique IDs to refetch objects when it determines that its cache is stale. (It uses the Query.node field to refetch objects.)

Defining UUIDs

You must provide a function for generating UUIDs and fetching objects with them. In your schema, define id_from_object and object_from_id:

 1MySchema = GraphQL::Schema.define do
 2  id_from_object ->(object, type_definition, query_ctx) {
 3    # Call your application's UUID method here
 4    # It should return a string
 5    MyApp::GlobalId.encrypt(object.class.name, object.id)
 6  }
 7
 8  object_from_id ->(id, query_ctx) {
 9    class_name, item_id = MyApp::GlobalId.decrypt(id)
10    # "Post" => Post.find(id)
11    Object.const_get(class_name).find(item_id)
12  }
13end

An unencrypted ID generator is provided in the gem. It uses Base64 to encode values. You can use it like this:

 1MySchema = GraphQL::Schema.define do
 2  # Create UUIDs by joining the type name & ID, then base64-encoding it
 3  id_from_object ->(object, type_definition, query_ctx) {
 4    GraphQL::Schema::UniqueWithinType.encode(type_definition.name, object.id)
 5  }
 6
 7  object_from_id ->(id, query_ctx) {
 8    type_name, item_id = GraphQL::Schema::UniqueWithinType.decode(id)
 9    # Now, based on `type_name` and `id`
10    # find an object in your application
11    # ....
12  }
13end

UUID fields

To participate in Relay’s caching and refetching, objects must do two things:

  • Implement the "Node" interface
  • Define an "id" field which returns a UUID

To implement the node interface, include GraphQL::Relay::Node.interface in your list of interfaces:

1PostType = GraphQL::ObjectType.define do
2  name "Post"
3  # Implement the "Node" interface for Relay
4  interfaces [GraphQL::Relay::Node.interface]
5  # ...
6end

To add a UUID field named "id", use the global_id_field helper:

1PostType = GraphQL::ObjectType.define do
2  name "Post"
3  # ...
4  # `id` exposes the UUID
5  global_id_field :id
6  # ...
7end

Now, PostType can participate in Relay’s UUID-based features.

node field (find-by-UUID)

You should also provide a root-level node field so that Relay can refetch objects from your schema. It is provided as GraphQL::Relay::Node.field, so you can attach it like this:

1QueryType = GraphQL::ObjectType.define do
2  name "Query"
3  # Used by Relay to lookup objects by UUID:
4  field :node, GraphQL::Relay::Node.field
5  # ...
6end
